# Changelog

## 2026-07-23

### Changed

- 软件更新页会校验打印导出组件的可执行文件与安装完整性；组件缺失或损坏时显示“需要修复”，即使已是最新版本也可主动重新安装。
- 明细识别合计与发票总额不一致改为“识别提醒”，不再标成严重问题或阻断材料齐备；历史条目会自动迁移。
- 设置把付款处理拆成两个独立开关：“付款截图 OCR”关闭后不再识别或套用截图金额，单独拖入截图时直接手动选择条目且不再显示“未识别到付款金额”；“实付默认等于发票金额”控制之后新建或批量导入的条目是自动填写发票金额还是留空待确认。
- 多选条目后可批量修改报账人，并为每条实际变化保留修改记录。

### Fixed

- 修复发布版打印导出组件漏打包 `models`、`builder` 及重依赖，导致拼接 PDF 时提示 `No module named 'tidoc_print.models'` 的问题。
- 发布流程新增打印组件成品自检，Windows / macOS 构建无法完整加载打印模块时不再发布。
- 修复组件文件已删除但软件更新页仍仅凭版本标记显示“已安装”的问题。
- 修复单独拖入或粘贴付款截图时错误沿用上一次批量导入范围，导致明明存在同金额条目却提示“没有金额相同的条目”的问题；Windows 打包版同时为 CSS / JS 资源增加版本标记，避免 WebView2 缓存继续运行旧匹配逻辑。
- 修复 Windows WebView2 把带查询参数的本地入口地址当作文件名，启动时显示“未找到文件”的问题；版本标记只保留在 CSS / JS 资源上。
- 修复 Windows 打包版条目较多时列表高度未正确收缩、无法滚动到底部的问题；材料绑定列表也增加独立滚动区域。
- Windows 付款截图 OCR 会先把手机长截图缩放到系统限制内，再识别完整图和顶部区域，避免超大图片直接识别失败。
- 修复单张发票 PDF 中的 20 位银行账号被误当成第二个发票号码，导致批量导入提示“包含 2 张发票”的问题。

## 2026-07-14

### Changed

- 打印 PDF 默认按条目连续排列发票、付款截图和查验单，旧式按材料类型分别拼接改为可选；条目编号与页码叠加仍默认开启，并可在生成前关闭。
- 自动发现更新弹窗精简重复版本文案，使用指南改为醒目的可折叠完整流程。
- 精简悬浮提示：短标题、销售方、备注和文件名不再重复弹出原文，仅在内容被截断时显示完整文字；状态原因、快捷键、路径和图标按钮说明继续保留。
- “已修改”统一只表示实付金额与发票总金额不一致；其他字段修改继续保留历史记录，不影响原有编辑逻辑。
- 批量拖入付款截图时按 OCR 金额自动绑定到唯一同额条目；识别失败、没有匹配或存在同金额条目时改为逐份手动选择，避免整批误绑到同一条目。
- 详情中删除附件或物品明细后改为原位更新，保留删除提示但不再关闭并重开弹窗。
- 拖拽和粘贴材料、付款截图 OCR、批量扫描与创建、绑定包导入和批量导出新增持续进度提示，后台处理中会显示当前阶段，不再长时间无反馈。

## 2026-07-13

### Fixed

- 修复部分数电发票 PDF 的税额行以 `¥` 结尾时，下一行账号被误识别为价税合计的问题。
- 补充小数点被空格拆散、明细名称跨行以及个人抬头发票的购买方 / 销售方识别兼容。
- 修复商品名在 PDF 首列跨行时只保留第一行的问题，并避免把规格型号续行误拼进商品名；批量导入会跳过包含多张发票的合并 PDF。
- 批量扫描同一发票号的重复 PDF / XML 时只保留一组，重复文件直接提示跳过，不再等到创建条目时失败。

## 2026-07-11

### Changed

- 选中条目的「批次」操作统一管理归属：可装入、补齐、移出，批次内还可原子移动到另一批次；右键单条也可快速处理。
- 标签操作会显示选中条目的已有情况，并新增全库标签重命名、合并和删除入口。
- 顶部关键词搜索支持发票金额和实付金额，并统一搜索、筛选、排序与批量操作的图标和控件层级。

### Fixed

- 修复条目装入批次后只有加入入口、无法从工作界面移出或移动的问题。
- 修复标签只能逐条移除、无法统一修正名称或彻底删除，以及标签筛选在重命名或删除后残留的问题。

## 2026-07-10

### Added

- 软件更新页新增 GitHub 图标与 GitHub Releases 手动下载入口，更新服务不可用时仍可下载安装包。
- 新增可选的启动后自动检查：默认关闭，每 24 小时最多联网一次，只检查版本，不自动下载或安装。
- 自动发现新版本和升级后的首次启动会显示 What’s changed，并在同一弹窗中提供快速开始工作流。
- 设置的高级数据维护可清理拖拽中转文件与旧更新包，同时保留材料、数据库、导出文件和待安装更新包。

### Changed

- 软件更新页改为紧凑的版本台账，突出当前版本、最新版本、包体积、版本变化和下一步动作，移除“当前平台”和更新清单地址等低价值信息。
- Windows 发布包改为按用户安装器，并将 PyInstaller 核心构建改为 onedir 后再压缩安装，降低日常启动时的一次性解包开销。
- GitHub Actions 自动从 CHANGELOG 最新一节生成客户端更新说明与 GitHub Release notes，再构建安装包、更新清单并发布到 COS / GitHub。

## 2026-07-09

### Added

- 支持拖拽 / 粘贴混合材料：发票进入导入预览，能按发票号唯一匹配的查验单会在创建条目后自动绑定，未匹配材料再手动选择条目。
- 付款截图添加时会用系统本地 OCR 尝试识别实付金额；单张可直接填入，多张会提示确认合计，已有手动实付金额时不覆盖。
- 批量创建条目接口返回创建结果明细，便于前端把后续材料绑定到新条目。

### Changed

- 设置里的「收款信息」收进「数据位置」区块，和本机数据、导出目录、迁移维护放在一起。
- 付款截图拼接 PDF 的叠加编号改为与发票 / 查验单一致的 `No.` / `Page` 风格。
- README 和设计文档补充当前材料绑定、付款截图 OCR 与轻量系统 OCR 的真实边界。

### Fixed

- macOS 上 Python 证书链无法验证更新源时，更新下载会改用系统信任链重试，避免已安装系统根证书但应用内 OpenSSL 仍报错。

## 2026-07-08

### Added

- 新增「报账批次」：运营组把一次要交的材料自由圈选、命名、留存为一个集合，可跨报账人、跨抬头、只取某人部分条目；主界面以文件夹式批次条显示条数与缺件数。
- 批次内每条可写「批次级催办备注」（如「张三缺查验单」），与条目自身的记账备注分离。
- 批次上下文条显示每报账人小计；导出 / 打印统一从选中条目的操作条进入。
- 条目支持打多个标签，并可批量打标 / 去标；高级筛选新增按条目标签、按有 / 无条目备注、按批次筛选。
- 主界面支持多栏条目列表、Shift 范围选择，并扩大复选框 / 抬头色条的选择热区，减少误打开详情。
- 报账人支持编辑姓名和审核人；设置新增「收款信息」用于打印 Word 的姓名、学号、电话、开户行、卡号。
- 设置里可把数据整体迁移到自定义位置，或恢复到系统默认位置。
- 设置新增启动默认抬头、默认列表密度偏好。

### Changed

- 设置页重构为配置中心（报账人、收款信息、偏好、数据位置、组件与更新、关于），移除与主界面重复的入口（批量导入 / 导入绑定包）和开发者视角的内部路径。
- 顶栏动作按主次重排：导入 / 设置归为次级，批量导入 / 新建为主操作；抬头、报账人、批次、状态、排序统一收到筛选工具条。
- 打印导出的发票 / 查验单拼接 PDF 改为右下角份数页码；付款截图改为横版 A4 每页两张，输出完成后可直接打开所在文件夹。
- 设置中的默认抬头和默认列表密度调整后会立即同步到主界面。
- 优化主界面字体、金额数字、条目选中态和首次使用提示的视觉层级。
- 数据库 schema 升级到 v2，历史库首次打开时自动补齐批次相关表。

## 2026-07-06

### Added

- 支持从文件夹或多选文件批量导入发票 PDF，并自动匹配同目录或子目录中的发票 XML。
- 支持将发票 PDF / XML 拖到主界面进入批量导入预览。
- 支持将付款截图、查验单拖到条目卡片或条目详情材料区绑定到指定条目。
- 新增首次使用提示，并在设置中提供“使用提示”入口。
- 新增拖拽中转区 `dropped/`，用于 WebView 拿不到本机路径时保存临时文件。

### Changed

- 批量导入只用发票 PDF 创建条目，XML 只作为辅助识别；付款截图和查验单不再按文件名自动绑定。
- 优化拖拽视觉反馈：空白列表区用于批量导入，条目卡片用于绑定材料，减少误拖。
- 设置页重组为常用操作、功能状态、本机数据，减少重复入口。
- 精简列表模式会收起次要信息和快捷按钮，而不只是压缩间距。
- 教育基金会条目只保留颜色和色条区分，不再额外强化显示抬头标签。

### Fixed

- 修复部分数电发票 PDF 文本流错位导致的购买方、销售方、明细和金额识别失败。
- 修复拖拽提示在 WebView 中可能不消失的问题。
- 修复 pywebview 文件对话框弃用警告，改用新版 `FileDialog` API。
- 过滤 macOS 上已知无害的 `NSSoftLinking` 输入法框架警告。
- 防止同一条目重复上传同一份附件。
- 防止将不同发票号的 PDF / XML 拖入已有条目造成误绑定。
